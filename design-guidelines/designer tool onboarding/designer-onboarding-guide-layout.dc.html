7Prompt 2 finding: Mark Throw's hover highlight, made touch-safe

Today, dragging the disc highlights whichever token the cursor is hovering over. Touch has no hover, and a finger dragging the disc also covers up whatever's underneath it. Fix: highlight by proximity to the drag point instead of exact hover, and offset the live preview above the fingertip so it's never hidden. Mouse drag keeps working exactly as it does today; touch gets the same feedback without needing hover at all.
